any, unknown এবং never এর মধ্যে পার্থক্য নিচে দেওয়া হলোঃ

  any হচ্ছে সবচেয়ে গ্রহণযোগ্য টাইপ এর ব্যাবহার করার ফলে টাইপ্সক্রিপ্ট এ ভেরিএবল এ যেকোন প্রকার ভেলু রাখা যায় যেমন নাম্বার, স্ট্রিং, বুলিয়ান টাইপ ডেটা। যেহেতু সব প্রকার ভেলু রাখা যায় এটার ফলে বড় প্রজেক্টে বাগ দেখা দিতে পারে জা কোড এর জন্য একদম ভাল না।

  unknow হচ্ছে আগের থেকে একটু নিরাপদ any তে যেহেতু যেকোনো ধরনের ডেটা রাখা যায় এবং তা সহজেই ব্যাবহার করা যায় সেই সমস্যা থেকে বাচার জন্য unknown এর ব্যাবহার করা হয় এতেও যেকোনো মান রাখা যায় তবে এটি ব্যাবহার করার আগে টাইপ টি চেক করে নিতে হয়।

  never হচ্ছে যা কখনই ঘটবে না, এমন কোন ফাংশন যদি থাকে যা কখনই কিছু রিটার্ন করবে না তখন আমরা সেটার টাইপ () নির্বাচন করতে পারি। এটাতে কোড কখনও পৌঁছাবে না।

    নিচে এদের কোড এর উদাহরণ দেওয়া হলঃ

        এখানে কোন ত্রুতি হবে না। 

            let value: any;

            value = 10;
            value = "Hello";
            value = true;

            value.doSomething();

    ভেলু যেকোনো কিছু রাখা যায় তবে আগে টাইপ চেক করে নিতে হবে।

        let data: unknown;

        data = 123;
        data = "Hello";

        if (typeof data === "string") {
        console.log(data.toUpperCase());
        }

    এটিতে কখনই পৌঁছাবে না।

        function throwError(message: string): never {
        throw new Error(message);
        }

Union এবং Intersection এর একটি উদাহরণ দিয়ে নিচে বুঝান হল।

ইউনিয়ন এর কাজ হল দুই বা ততধিক শর্তের মধ্যে যেকোনো একটি হতে পারে। মানে একটি ভেরিয়েবল এর মধ্যে একাধিক টাইপের যেকোনো একটি হতে পারে। এটি "।" দ্বারা প্রকাশ করা হয়। এর মানে হল যেই টাইপ গুলি দেওয়া থাকবে তারা সবাই বৈধ।

অপরদিকে Intersection হল একটি এর মধ্যে একাধিক কে সমন্বয় করা অর্থাৎ একটি টাইপের মধ্যে সব গুলর বৈশিষ্ট্য থাকতে হবে। এটিকে "&" দ্বারা প্রকাশ করা হয়।

    type User = { name: string };
    type Admin = { role: "admin" };
    type Member = { role: "member" };

    // Union
    type UserRole = Admin | Member;

    // Intersection
    type AdminUser = User & Admin;

    const u1: UserRole = { role: "admin" };
    const u2: UserRole = { role: "member" };

    const admin: AdminUser = {
    name: "Asha",
    role: "admin"
    };

উপরের উদাহরণ এ ইউজার মেম্বার ও হতে পারবে এডমিন ও হতে পারবে দুটি বৈধ । কিন্তু যদি এডমিন হয় তাহলে তার সাধারন তথ্যের সাথে এডমিন এর বিশেষ তথ্য থাকা লাগবে।
